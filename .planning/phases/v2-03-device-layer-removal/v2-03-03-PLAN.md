  <action>
    There are 3 wait_ready functions to extract. Each currently takes a bus struct pointer and reads from subscribers embedded in that struct. The extraction adds a standalone version that takes only the subscriber handle(s) and data pointer(s), while keeping the original bus-based function as a thin wrapper (so existing callers don't break).

    **1. observe_topics — `platform_observe_bus_wait_ready`**

    Current signature: `void platform_observe_bus_wait_ready(platform_observe_bus_t *bus, platform_ins_state_message_t *ins_msg)`
    Current body: polls `bus->ins_sub` until `ins_msg->ready != 0`.

    Add standalone function in `observe_topics.c`:
    ```c
    void platform_observe_wait_ins_ready(Subscriber_t *ins_sub, platform_ins_state_message_t *ins_msg)
    {
        while (ins_msg->ready == 0U)
        {
            SubGetMessage(ins_sub, ins_msg);
            osDelay(1);
        }
    }
    ```
    Declare in `observe_topics.h`.
    Rewrite the original as a wrapper:
    ```c
    void platform_observe_bus_wait_ready(platform_observe_bus_t *bus, platform_ins_state_message_t *ins_msg)
    {
        platform_observe_wait_ins_ready(bus->ins_sub, ins_msg);
    }
    ```

    **2. actuator_topics — `platform_actuator_bus_wait_ready`**

    Current signature: `void platform_actuator_bus_wait_ready(platform_actuator_bus_t *bus, platform_ins_state_message_t *ins_msg)`
    Current body: polls `bus->ins_sub` until `ins_msg->ready != 0`.

    Add standalone function in `actuator_topics.c`:
    ```c
    void platform_actuator_wait_ins_ready(Subscriber_t *ins_sub, platform_ins_state_message_t *ins_msg)
    {
        while (ins_msg->ready == 0U)
        {
            if (SubGetMessage(ins_sub, ins_msg))
            {
                if (ins_msg->ready != 0U)
                {
                    break;
                }
            }
            osDelay(1);
        }
    }
    ```
    Declare in `actuator_topics.h`.
    Rewrite the original as a wrapper:
    ```c
    void platform_actuator_bus_wait_ready(platform_actuator_bus_t *bus, platform_ins_state_message_t *ins_msg)
    {
        platform_actuator_wait_ins_ready(bus->ins_sub, ins_msg);
    }
    ```

    **3. chassis_topics — `chassis_runtime_bus_wait_ready`**

    Current signature: `void chassis_runtime_bus_wait_ready(Chassis_Runtime_Bus_t *bus, platform_ins_state_message_t *ins, platform_device_feedback_t *feedback)`
    Current body: polls multiple subscribers until `ins->ready != 0 && feedback->actuator_feedback.valid`.

    Add standalone function in `chassis_topics.c`:
    ```c
    void chassis_wait_inputs_ready(Chassis_Runtime_Bus_t *bus,
                                   platform_ins_state_message_t *ins,
                                   platform_device_feedback_t *feedback)
    {
        while (ins->ready == 0U || !feedback->actuator_feedback.valid)
        {
            platform_robot_intent_t intent = {0};
            platform_chassis_observe_message_t observe = {0};
            chassis_runtime_bus_pull_inputs(bus, ins, &intent, &observe, feedback);
            osDelay(1);
        }
    }
    ```
    Declare in `chassis_topics.h`.
    Rewrite the original as a wrapper:
    ```c
    void chassis_runtime_bus_wait_ready(Chassis_Runtime_Bus_t *bus,
                                        platform_ins_state_message_t *ins,
                                        platform_device_feedback_t *feedback)
    {
        chassis_wait_inputs_ready(bus, ins, feedback);
    }
    ```

    Note: The chassis wait_ready is harder to fully decouple from the bus struct because it calls `chassis_runtime_bus_pull_inputs` which reads 4 subscribers. For now, the standalone function still takes the bus struct but has a name that doesn't imply "bus_wait_ready" — this is sufficient for Phase 4 to refactor further. The key requirement is that the wait logic is identifiable as a standalone concern.
