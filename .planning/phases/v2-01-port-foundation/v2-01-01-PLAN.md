<objective>
Unify `platform_actuator_command_t` and `platform_device_command_t` into a single canonical command type using indexed form (`joints[N]`, `wheels[N]`), eliminating ~160 lines of field-by-field mapping code in actuator_gateway.c and device_layer.c.

Purpose: Satisfies SLIM-04. Removes the redundant parallel command types that force every command to be copied field-by-field between control and device layers. This is a pure refactor — zero behavioral change.

Output: One command type used end-to-end from balance_controller through actuator_gateway to device_layer. All mapping functions deleted. All 11 CTest targets pass unchanged.
</objective>

    **Step 1: Restructure actuator_command.h to use indexed form**

    Replace `platform_motor_command_set_t` definition from named fields:
    ```c
    typedef struct {
      platform_motor_command_t left_leg_joint[2];
      platform_motor_command_t right_leg_joint[2];
      platform_motor_command_t left_wheel;
      platform_motor_command_t right_wheel;
    } platform_motor_command_set_t;
    ```
    With indexed form:
    ```c
    typedef struct {
      platform_motor_command_t joints[PLATFORM_JOINT_MOTOR_COUNT];
      platform_motor_command_t wheels[PLATFORM_WHEEL_MOTOR_COUNT];
    } platform_motor_command_set_t;
    ```

    Add a named accessor enum in `actuator_command.h` (after the struct, before `platform_actuator_command_t`) so robot-profile code can use meaningful names:
    ```c
    enum {
      PLATFORM_JOINT_LEFT_FRONT  = 0,
      PLATFORM_JOINT_LEFT_REAR   = 1,
      PLATFORM_JOINT_RIGHT_FRONT = 2,
      PLATFORM_JOINT_RIGHT_REAR  = 3,
    };

    enum {
      PLATFORM_WHEEL_LEFT  = 0,
      PLATFORM_WHEEL_RIGHT = 1,
    };
    ```

    **Step 2: Delete device_command.h or make it a thin alias**

    `platform_device_command_t` in `device_command.h` currently has `platform_motor_device_command_t joints[4]` and `wheels[2]` — a DIFFERENT per-motor struct with extra `device_id` and `kind` fields. After unification, `platform_device_command_t` is no longer needed as a separate type.

    However, `device_command.h` is included by `actuator_device.h` and `device_layer.h`. The safest approach: keep `device_command.h` but redefine `platform_device_command_t` as a typedef alias for `platform_actuator_command_t`, OR replace all usages of `platform_device_command_t` with `platform_actuator_command_t` directly.

    Preferred approach: Replace `platform_device_command_t` everywhere with `platform_actuator_command_t`. Update `device_command.h` to just `#include "actuator_command.h"` and add `typedef platform_actuator_command_t platform_device_command_t;` as a temporary compatibility alias. This avoids touching every include site in one shot.

    The `platform_motor_device_command_t` struct (with `device_id`, `kind` fields) is ONLY used inside `actuator_gateway.c` mapping code and `test_actuator_gateway.c` assertions. Once the mapping is deleted, this struct is dead. Remove it from `device_command.h`.
