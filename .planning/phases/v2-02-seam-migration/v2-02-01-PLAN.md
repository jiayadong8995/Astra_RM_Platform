Create `ports_fake.h` header declaring a test injection API that mirrors the 5 operations currently provided by device_layer test hooks:

```c
#ifndef PLATFORM_BSP_PORTS_FAKE_H
#define PLATFORM_BSP_PORTS_FAKE_H

#include "ports.h"

typedef platform_device_result_t (*platform_ports_fake_imu_hook_t)(platform_imu_sample_t *sample, void *context);
typedef platform_device_result_t (*platform_ports_fake_remote_hook_t)(platform_rc_input_t *input, void *context);
typedef platform_device_result_t (*platform_ports_fake_feedback_hook_t)(platform_device_feedback_t *feedback, void *context);
typedef platform_device_result_t (*platform_ports_fake_command_hook_t)(const platform_motor_command_set_t *cmd, void *context);

typedef struct {
    platform_ports_fake_imu_hook_t read_imu;
    platform_ports_fake_remote_hook_t read_remote;
    platform_ports_fake_feedback_hook_t read_feedback;
    platform_ports_fake_command_hook_t write_command;
    void *context;
} platform_ports_fake_hooks_t;

void platform_ports_fake_set_hooks(const platform_ports_fake_hooks_t *hooks);
void platform_ports_fake_reset_hooks(void);

#endif
```

Key design notes:
- No `init` hook needed — the old `init_default_profile` hook was only used by `test_device_profile_safety_seams.c` to count init calls. The port interface has no init function, so this hook is dropped. The seams test will be rewritten to verify port function injection directly.
- `write_command` takes `const platform_motor_command_set_t *cmd` (matching `ports.h` signature), NOT `const platform_device_command_t *command` (the old device_layer signature). Tests that captured `platform_device_command_t` will need minor adjustment — the `platform_motor_command_set_t` is the `.motors` field of `platform_device_command_t`.
- Hook function pointer signatures use `(output_ptr, void *context)` pattern matching the old hooks.

Rewrite `ports_fake.c` to:
1. Include `ports_fake.h`
2. Store hooks in a static `g_ports_fake_hooks` variable
3. Each port function checks if the corresponding hook is non-NULL; if so, delegates to hook; otherwise returns zeroed OK (preserving current default behavior)
4. Implement `platform_ports_fake_set_hooks` and `platform_ports_fake_reset_hooks`

Verify the file compiles by checking existing `test_bsp_ports_compile` still passes (it links ports_fake.c directly).
