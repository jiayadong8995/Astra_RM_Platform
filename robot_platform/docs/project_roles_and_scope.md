# Project Roles And Scope

这份文档定义当前项目的协作分工、交付范围和决策边界。

目的只有一个：

- 让架构、仿真、重构、决策四条线在同一套口径下推进
- 避免不同角色各自定义目标、接口和完成标准

## 1. 项目协作角色

当前项目按四个角色推进：

1. 决策者
2. 总架构师
3. sim 编码师
4. 代码重构师

## 2. 各角色职责

### 2.1 决策者

职责：

- 决定当前阶段做什么、不做什么
- 决定资源投入优先级
- 对关键路线和接口拥有最终拍板权

当前重点：

- 保持项目目标收敛
- 避免任务发散到非当前主线

### 2.2 总架构师

职责：

- 定义项目统一口径
- 定义平台目标架构和当前边界
- 审核各执行线的方案是否一致
- 收口跨模块、跨角色的接口定义

当前重点：

- 保证 `robot_platform` 是唯一主执行面
- 保证 `SITL` 与 runtime 重构使用同一套边界
- 避免 sim 线和重构线各自发明接口

### 2.3 sim 编码师

职责：

- 负责平台验证侧基础设施
- 负责 `SITL`、`bridge` 和最小验证运行入口的实现
- 负责验证平台化之后控制链行为是否保持正确

交付目标：

- 一套可运行的验证平台
- 能承载当前控制链运行
- 能验证迁移后的业务逻辑没有明显跑偏

注意：

- sim 线不负责定义业务架构
- sim 线不负责决定 runtime 分层
- sim 线不负责单独定义控制链接口

### 2.4 代码重构师

职责：

- 负责把 legacy 业务代码逐步收进统一平台结构
- 负责收紧 `generated / bsp / module / app` 的边界
- 负责减少 legacy 头文件、全局变量、板级细节的直接耦合

交付目标：

- 把原有底盘控制链收进 `robot_platform`
- 让同一条主控制链同时服务 `hw` 和 `sitl`
- 让平台结构逐步从“legacy 落位”走向“真正的平台分层”

注意：

- 重构线不负责另起一套验证体系
- 重构线不负责自己定义外部验证协议
- 重构线不负责改变总体架构方向

## 3. 当前两条执行主线

当前执行面只有两条主线：

1. sim 验证线
2. runtime 重构线

它们的关系是：

- 重构线负责把控制链搬进平台
- sim 线负责证明这条控制链在平台里跑得对

这两条线不是并行独立产品线，而是同一个平台项目里的两个执行方向。

## 4. 当前交付定义

### 4.1 sim 线交付定义

sim 线的交付不是“做一个看起来能跑的仿真 demo”，而是：

一套可运行的验证平台，用来验证平台化之后，业务逻辑和控制链行为仍然正确。

当前可接受的交付范围：

- `SITL` 运行入口
- `bridge`
- 基本结果输出

当前不要求：

- `replay`
- 高保真动力学
- 场景库
- `physics_sim`

### 4.2 重构线交付定义

重构线的交付不是“重写一遍业务代码”，而是：

把现有 legacy 底盘控制代码逐步收进统一机器人平台里的运行时资产。

当前可接受的交付范围：

- 收进 `generated / bsp / module / app`
- 收紧依赖边界
- 减少 legacy 直连
- 保证 `hw` 与 `sitl` 能共享主控制链
- 推进新消息总线体系全面落地

当前不要求：

- 大范围算法重写
- 新 OS 抽象层

当前阶段的直接工作项：

- 清理无效构建入口和历史残留 target
- 稳定 `hw_elf` 与 `sitl` 共享的主控制链
- 继续推进新消息总线替代全局变量直连
- 收紧 `module / app / bsp` 与 `main.h`、板级头文件的耦合

## 5. 当前优先级

### P0

- runtime 重构主线
- 新消息总线体系落地
- SITL 稳定化

### P1

- flash
- debug
- test 统一入口
- replay

说明：

- `flash / debug / test` 需要在平台主链稳定后再收口
- `replay` 保留在 sim 线，但后置到当前交付之后

## 6. 接口决策原则

这是当前项目最重要的协作规则。

所有跨线接口定义，必须由：

- 总架构师
- 决策者

共同收口后才能进入实现。

### 6.1 必须统一决策的接口

- replay 输入输出格式
- SITL bridge 的输入输出格式
- 板上日志格式
- runtime 中 app/module/bsp 的边界数据结构
- 后续面向 `micro-ROS` 的对外消息语义

### 6.2 不能各自定义的内容

sim 编码师和代码重构师不能各自单独定义：

- 一套自己的消息结构
- 一套自己的日志协议
- 一套自己的控制链 I/O 边界

否则最后会出现两边都“完成实现”，但系统接不起来。

## 7. 当前项目共识

当前项目的统一共识应当是：

- 这是一个平台架构项目，不是多个并列子项目
- `robot_platform` 是唯一主执行面
- `Astra_RM2025_Balance` 是历史基线
- `references` 是外部参考集
- 当前验证主线是 `SITL`
- 新消息总线体系进入当前交付面
- `replay`、`physics_sim`、`osal`、大范围算法重写都不进入当前执行面

## 8. 当前阶段的一句话定义

当前阶段的目标可以压缩成一句话：

在统一接口决策前提下，由重构线把 legacy 底盘控制链和新消息总线体系收进 `robot_platform`，由 sim 线提供可运行的 `SITL` 验证平台，并先完成最小平台闭环。
