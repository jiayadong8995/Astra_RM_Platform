# app_flow

`app_flow` 现在只保留应用级业务编排。

当前这一层仍然有用，但职责已经收窄：

- 把 `app_io` 收到的输入整理成业务语义
- 生成 `RobotIntent` 或项目级命令
- 处理模式切换、行为请求、项目级限幅

它不再负责：

- 状态形成
- 平衡控制求解
- 执行器下发
- 板级访问

对 `balance_chassis` 来说，这一层当前主要承接：

- `remote_orchestration.*`

也就是：

- `rc/姿态/状态摘要 -> 机器人意图/底盘命令`

如果后面 app 层继续收缩，这一层可以继续保留为“模式与意图编排层”，
也可以进一步改名为 `modes/` 或 `intent/`。
